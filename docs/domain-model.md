# ChaosForge – Domain Model Reference

## Entity Overview

```
Project
  ├── UseCase[]
  │     └── URS[]  (User Requirements Specification)
  │           └── SRS[]  (Software Requirements Specification)
  │                 └── WorkTask[]
  │                       └── TaskAttempt[]
  ├── AgentSlot[]          (configuration: role + count)
  ├── AgentInstance[]      (running agent instances)
  ├── Sprint[]
  └── RevisionGate[]       (BA / Architect / ScrumMaster revision gates)
```

## Enumerations

### ProjectStatus
```
Setup → RequirementsPhase → ArchitecturePhase → SprintPlanning → Development → Completed
```

### WorkTaskStatus
```
Backlog → InProgress → InReview → InTesting → InDocumentation → Done
                                      ↓               ↓
                                   Backlog          Backlog   (on rejection)
```

### AgentRole
```
BusinessAnalyst  (singleton)
Architect        (singleton)
ScrumMaster      (singleton)
Developer        (1..N)
Tester           (1..N)
Reviewer         (1..N)
TechnicalWriter  (1..N)
```

### AttemptType
`Development | Review | Testing | Documentation`

### AttemptResult
`Pending | Approved | Rejected`

### RevisionGateType
`AfterBA | AfterArchitect | AfterScrumMaster`

### RevisionGateAction
`Accept | EditAndAccept | Reject`

### AgentInstanceStatus
`Idle | Working | Blocked | Finished`

---

## Key Invariants (enforced at domain level)

- `AgentSlot.Count` must be 1 for `BusinessAnalyst`, `Architect`, `ScrumMaster`
- A `WorkTask` can only have one `TaskAttempt` with `Result = Pending` at a time
- `RevisionGate.RejectionReason` is required when `Action = Reject`
- `RevisionGate.HumanEditedOutput` is required when `Action = EditAndAccept`
- A `WorkTask` can only move to `InProgress` if its `Sprint` is active
